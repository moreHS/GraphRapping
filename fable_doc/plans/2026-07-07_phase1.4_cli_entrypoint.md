## CLI 설계

`src/cli.py` — argparse subparsers, 신규 의존성 없음.

```
graphrapping migrate [--dsn DSN]
graphrapping full-load [--dsn DSN] [--review-json PATH] [--product-json PATH]
                        [--user-profiles-json PATH]
                        [--source-review-stats-json PATH]
                        [--kg-mode {off,on}] [--skip-validation]
graphrapping incremental [--dsn DSN] [--product-json PATH]
                          [--batch-size N] [--kg-mode {off,on}]
                          [--skip-validation]
graphrapping validate [--dsn DSN]
                       [--expected-min-active-products N] ... (validate_all
                       kwargs 중 실사용 가치 있는 것만 노출)
```

옵션 기본값은 `scripts/run_906_full_load_db.py`의 기본값을 그대로 계승
(mockdata 경로 등) — 로컬/데모 환경에서 인자 없이도 실행 가능하게.

공통:
- `--dsn` 기본값 = `resolve_database_url()` 지연 평가 (인자 없으면 env로
  fallback, 둘 다 없으면 실행 시점에 에러 메시지 출력 후 exit 1 — argparse
  파싱 단계가 아니라 실행 단계에서 판단해야 `--help`가 env 유무와 무관하게
  항상 동작함).
- 각 서브커맨드 핸들러는 async 함수, `asyncio.run()`으로 래핑.
- 예외 발생 시 traceback 대신 `f"error: {exc}"` 한 줄 출력 + exit 1.
- pool은 커맨드 실행마다 `asyncpg.create_pool(dsn, min_size=1, max_size=2)`
  생성 후 `finally`에서 `close()`.
- `validate` 서브커맨드는 결과 요약을 사람이 읽을 수 있게 출력 (각 check의
  name/status/message, 마지막에 전체 status) 후 `INVALID`면 exit 1.
