# [초안] 한국어 임베딩 모델 사용 승인 신청 — GraphRapping 사전 성장 보조 (3.2)

작성: 2026-07-10 · 상태: **초안 (사내 승인 양식으로 이관용)** · 근거 계획:
fable_doc/03_improvement_plan.md §3.2

> 이 문서는 사내 보안/정책 승인 프로세스에 제출할 내용의 초안이다. 실제 양식에
> 맞춰 옮겨 쓰되, "사용 범위" 절의 제약 5개는 검토 표면을 줄이는 핵심이므로
> 그대로 유지할 것을 권장한다.

## 1. 신청 목적

리뷰 파이프라인의 사전(concept dictionary) 유지보수 병목 해소.
`quarantine_unknown_keyword`(사전에 없는 신규 키워드 격리 테이블)의 항목을
사람이 전수 검토해 사전 등록 여부를 판단하는 현행 수작업 루프에, **기존 concept
중 의미가 가까운 top-k 후보를 자동 제안하는 리포트**를 추가해 검토 속도를
높인다. 판단과 반영은 계속 사람이 한다.

## 2. 사용 범위 (제약 — 승인 범위의 경계)

1. **오프라인 배치 전용** — 서빙/추천 런타임에 포함되지 않음 (런타임 의존성 0)
2. **로컬 추론** — 사내 장비에서 모델 가중치를 내려받아 실행. 외부 API 호출·
   데이터 외부 전송 없음
3. **입력 데이터 최소화** — 리뷰 원문이 아니라 quarantine된 **키워드 표면형**
   (짧은 한국어 단어/구)과 사내 사전의 concept 라벨만 임베딩. 개인정보·원문
   텍스트 미포함
4. **스코어링/추천 미개입** — 추천 점수·후보 생성에 임베딩을 사용하지 않음
   (evidence-first 계약 유지). 산출물은 사전 관리자용 제안 리포트뿐
5. **사람 승인 게이트** — 제안의 자동 반영 금지. 승인된 항목만 사전에 등록

## 3. 후보 모델 (전부 로컬 실행 오픈소스 — 최종 1종은 승인 범위 내 선정)

| 모델 | 라이선스(제출 전 재확인) | 비고 |
|---|---|---|
| `nlpai-lab/KURE-v1` | MIT | 한국어 retrieval 특화 |
| `snunlp/KR-SBERT-V40K-klueNLI-augSTS` | 공개(재확인 필요) | 한국어 SBERT 표준형 |
| `jhgan/ko-sroberta-multitask` | 공개(재확인 필요) | 경량 대안 |

실행 스택: `sentence-transformers` (Apache-2.0) + 로컬 가중치 캐시.

## 4. 산출물과 운영

- 배치 리포트: `quarantine_unknown_keyword` 각 항목 → 기존 concept top-k
  (유사도 점수 포함) 제안 목록
- 소비처: `src/qa/dictionary_growth.py` 사전 성장 루프의 검토 입력
- 보관: 배치 산출물로 리포지토리/아티팩트 보관 (모델 출력이 DB에 직접
  쓰이지 않음)

## 5. 리스크 자체 평가

| 항목 | 평가 |
|---|---|
| 데이터 유출 | 없음 — 로컬 추론, 외부 전송 없는 구조. 입력도 키워드 표면형뿐 |
| 모델 편향/오류 영향 | 제안 단계에 한정 — 오제안은 사람 게이트에서 기각됨. 추천 결과에 직접 영향 없음 |
| 운영 영향 | 없음 — 오프라인 배치 전용, 서빙 경로 의존성 추가 없음 (optional dependency) |
| 라이선스 | 후보 전부 오픈소스. 최종 선정 시 라이선스 문구 재확인 항목 명시 |

## 6. 요청 항목

1. 위 사용 범위(§2) 하에서의 한국어 문장 임베딩 모델 사용 승인
2. 오프라인 배치용 python 의존성(`sentence-transformers` 계열) 추가 승인
   (pyproject optional extra로 격리 — 미설치 환경 동작 무영향)
